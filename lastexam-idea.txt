

[ 바이브코딩 - 기말고사 ]

1. 주제 : 증권사 이상금융거래 탐지시스템 시라니오 자동 생성 시스템 구축
2. 핵심 내용 : 이상금융 거래 탐지에 활용할 수 있는 3개의 조건을 입력하면, 자동으로 해당 이상징후를 탐지할 수 있는 시나리오를 만들고, 이를 기반으로 수집된 거래내역 테이블에서 이를 탐지할 수 있는 "SQL"을 생성해주는 프로그램
3. 핵심 기능
  가. 입력 화면 : 이상금융거래 탐지 조건 3개의 키워드 입력
  나. 출력 화면1 : 이상금융거래 탐지 시나리오 3개 제시, 각 시나리오의 조건별 임계치 표시. 각 시나리오 하단에 이상금융거래 탐지 SQL 까지 작성하여 제시, 화면 구성은 "사나리오 1~3"으로 명시하고, 각 시나리오 하단에 SQL문장을 표시하도록 구성하고, 각 시나리오별 선택버튼을 두어 이를 선택하면 이상금융거래 탐지 시나리오 테이블에 저장할 수 있도록 화면구성
  다. 출력 화면2 : 등록된 이상금유거래 탐지 시나리오를 실행시켜 실제 탐지 결과를 리스트 형태로 조회하는 화면 구성
4. 테이블 구성
  가. 고객 정보 테이블 : 고객 ID, 고객나이, 고객성별, 고객주거지역, 고객 등록일자
  나. 고객 거래정보 테이블 : 고객 ID, 거래번호, 출금일자, 출금시각, 출금금액, 출금매체
  다. 이상금융거래 탐지 시나리오 테이블 : 시나리오ID, 시나리오 등록일, 등록시간, 조건별 임계치 숫자
  라. 실제 탐지 결과 내역 테이블 : 고객ID, 거래번호, 시나리오 ID
5. 합성 데이터 구성
  가. 고객 정보 테이블 : 0~80세이 이르는 고객정보 500개를 자동으로 생성하여 입력
  나, 고객 거래정보 테이블 : 상기 고객정보 테이블에 생성된 고객 데이터를 기반으로 하여 10000개의 거래내역을 생성하여 입력
